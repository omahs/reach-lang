class=\"icon icon-link\"></span></a></h2>\n<p>\n  <i id=\"p_1\" class=\"pid\"></i>The <a href=\"https://www.algorand.com/\" target=\"_blank\">Algorand</a> Reach connector generates a\n  contract that manage one instance of the DApp's\n  execution.<a href=\"#p_1\" class=\"pid\">1</a>\n</p>\n<p>\n  <i id=\"p_2\" class=\"pid\"></i>It uses finite on-chain state.\n  The DApp consists of one application.\n  The contract escrow account is the application account.\n  Sometimes a companion contract is created that is used to increase the opcode budget during contract execution.<a href=\"#p_2\" class=\"pid\">2</a>\n</p>\n<p>\n  <i id=\"p_3\" class=\"pid\"></i>It relies on versions of <code>algod</code> that support TEAL version 6, such as Algorand 3.5.1.\n  It uses the Algorand <code>indexer</code> version 2 to lookup and monitor publications; in other words, it does <em>not</em> rely on any communication network other than Algorand itself.<a href=\"#p_3\" class=\"pid\">3</a>\n</p>\n<p>\n  <i id=\"p_4\" class=\"pid\"></i>Algorand uses the SHA256 algorithm to perform digests.\n  Its bit width is 64-bits.<a href=\"#p_4\" class=\"pid\">4</a>\n</p>\n<p>\n  <i id=\"p_5\" class=\"pid\"></i>Non-network tokens are compiled to <a href=\"https://developer.algorand.org/docs/features/asa/\" target=\"_blank\">Algorand Standard Assets</a> (ASAs).\n  Specifically, the <span class=\"snip\"><a href=\"/rsh/compute/#rsh_Token\" title=\"rsh: Token\"><span style=\"color: var(--shiki-color-text)\">Token</span></a></span> type refers to the id of the ASA.<a href=\"#p_5\" class=\"pid\">5</a>\n</p>\n<p>\n  <i id=\"p_6\" class=\"pid\"></i>Token minting creates an ASA owned and managed by the contract account.\n  Freezing, clawback, reserves, and separate managers are not supported.<a href=\"#p_6\" class=\"pid\">6</a>\n</p>\n<p>\n  <i id=\"p_7\" class=\"pid\"></i>Views are compiled to client-side functions that can interpret the global and local state of the Algorand Application associated with the DApp.\n  This means they are sensitive to the particular compilation details of the particular Reach program.\n  We hope to work with the Algorand community to define a standard for views.\n  Views expand the on-chain state to include the free variables of all values bound to a view.<a href=\"#p_7\" class=\"pid\">7</a>\n</p>\n<p><i id=\"p_8\" class=\"pid\"></i><span class=\"snip\"><a href=\"/rsh/consensus/#rsh_Map\" title=\"rsh: Map\"><span style=\"color: var(--shiki-token-constant)\">Map</span></a></span>s on Algorand only support keys that are of type <span class=\"snip\"><a href=\"/rsh/compute/#rsh_Address\" title=\"rsh: Address\"><span style=\"color: var(--shiki-color-text)\">Address</span></a></span>.<a href=\"#p_8\" class=\"pid\">8</a></p>\n<p>\n  <i id=\"p_9\" class=\"pid\"></i>Linear state is compiled into Application Local State.\n  This means that participants must explicitly \"opt-in\" to storing this state on their account (which increases their minimum balance).\n  The Reach standard library will do this automatically when connecting to Reach generated contracts, but other users must be specifically programmed to do this.\n  This \"opt-in\" requirement means that DApps with linear state deployed on Algorand can deadlock and be held hostage:\n  Suppose that Alice transfers 10 ALGO to a contract in step one, then in step two, the consensus must store a value associated with Bob, and then she can receive her 10 ALGO back, then the program terminates.\n  On some networks, Alice can perform these two steps completely on her own and she is in complete control of her funds.\n  However, on Algorand, running this program requires that Bob \"opt-in\" to storing values for the application.\n  We hope that future versions of Algorand will allow other parties to pay the fees to \"opt-in\" to applications to prevent these kinds of deadlock attacks.<a href=\"#p_9\" class=\"pid\">9</a>\n</p>\n<p>\n  <i id=\"p_10\" class=\"pid\"></i>In Algorand, network time corresponds to round numbers.\n  Each round is assigned a Unix timestamp, but when you look at the timestamp in code executing in round N+1, you read the timestamp assigned to round N.\n  Ensure that you read <a href=\"/rsh/errors/#RW0006\">RW0006</a> if you use network seconds.<a href=\"#p_10\" class=\"pid\">10</a>\n</p>\n<p><i id=\"p_11\" class=\"pid\"></i>The connector provides a binding named <span class=\"snip\"><a href=\"/rsh/appinit/#rsh_ALGO\" title=\"rsh: ALGO\"><span style=\"color:
