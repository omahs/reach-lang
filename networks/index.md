[{"bookPath":"","title":"Networks","titleId":"ref-networks","hasOtp":true,"hasPageHeader":true},"<h1 id=\"ref-networks\" class=\"refHeader\">Networks<a aria-hidden=\"true\" tabindex=\"-1\" href=\"#ref-networks\"><span class=\"icon icon-link\"></span></a></h1>\n<p>\n  <i id=\"p_0\" class=\"pid\"></i>This section describes the consensus network connectors\n  supported by Reach version 0.1.13:<a href=\"#p_0\" class=\"pid\">0</a>\n</p>\n<ul>\n  <li><a href=\"/networks/#ref-network-algo\">Algorand</a></li>\n  <li><a href=\"/networks/#ref-network-eth\">Ethereum</a></li>\n</ul>\n<h2 id=\"ref-network-algo\" class=\"refHeader\">Algorand<a aria-hidden=\"true\" tabindex=\"-1\" href=\"#ref-network-algo\"><span class=\"icon icon-link\"></span></a></h2>\n<p>\n  <i id=\"p_1\" class=\"pid\"></i>The <a href=\"https://www.algorand.com/\" target=\"_blank\">Algorand</a> Reach connector generates a\n  contract that manage one instance of the DApp's\n  execution.<a href=\"#p_1\" class=\"pid\">1</a>\n</p>\n<p>\n  <i id=\"p_2\" class=\"pid\"></i>It uses finite on-chain state.\n  The DApp consists of one application.\n  The contract escrow account is the application account.\n  Sometimes a companion contract is created that is used to increase the opcode budget during contract execution.<a href=\"#p_2\" class=\"pid\">2</a>\n</p>\n<p>\n  <i id=\"p_3\" class=\"pid\"></i>It relies on versions of <code>algod</code> that support TEAL version 6, such as Algorand 3.5.1.\n  It uses the Algorand <code>indexer</code> version 2 to lookup and monitor publications; in other words, it does <em>not</em> rely on any communication network other than Algorand itself.<a href=\"#p_3\" class=\"pid\">3</a>\n</p>\n<p>\n  <i id=\"p_4\" class=\"pid\"></i>Algorand uses the SHA256 algorithm to perform digests.\n  Its bit width is 64-bits.<a href=\"#p_4\" class=\"pid\">4</a>\n</p>\n<p>\n  <i id=\"p_5\" class=\"pid\"></i>Non-network tokens are compiled to <a href=\"https://developer.algorand.org/docs/features/asa/\" target=\"_blank\">Algorand Standard Assets</a> (ASAs).\n  Specifically, the <span class=\"snip\"><a href=\"/rsh/compute/#rsh_Token\" title=\"rsh: Token\"><span style=\"color: var(--shiki-color-text)\">Token</span></a></span> type refers to the id of the ASA.<a href=\"#p_5\" class=\"pid\">5</a>\n</p>\n<p>\n  <i id=\"p_6\" class=\"pid\"></i>Token minting creates an ASA owned and managed by the contract account.\n  Freezing, clawback, reserves, and separate managers are not supported.<a href=\"#p_6\" class=\"pid\">6</a>\n</p>\n<p>\n  <i id=\"p_7\" class=\"pid\"></i>Views are compiled to client-side functions that can interpret the global and local state of the Algorand Application associated with the DApp.\n  This means they are sensitive to the particular compilation details of the particular Reach program.\n  We hope to work with the Algorand community to define a standard for views.\n  Views expand the on-chain state to include the free variables of all values bound to a view.<a href=\"#p_7\" class=\"pid\">7</a>\n</p>\n<p>\n  <i id=\"p_8\" class=\"pid\"></i>Linear state is compiled into application box storage.\n  The names of boxes are either:<a href=\"#p_8\" class=\"pid\">8</a>\n</p>\n<ul>\n  <li><i id=\"p_9\" class=\"pid\"></i><code>${MapIndexByte}${KeyBytes}</code>, where <code>MapIndexByte</code> is the single-byte representation of which <span class=\"snip\"><a href=\"/rsh/consensus/#rsh_Map\" title=\"rsh: Map\"><span style=\"color: var(--shiki-token-constant)\">Map</span></a></span> it is and <code>KeyBytes</code> is the ABI encoding of the key, if this is less than 64 bytes; or<a href=\"#p_9\" class=\"pid\">9</a></li>\n  <li><i id=\"p_10\" class=\"pid\"></i><code>sha256(${MapIndexBytes}${KeyBytes})</code>, where <code>MapIndexBytes</code> is the multi-byte representation of the <span class=\"snip\"><a href=\"/rsh/consensus/#rsh_Map\" title=\"rsh: Map\"><span style=\"color: var(--shiki-token-constant)\">Map</span></a></span> it is, otherwise.<a href=\"#p_10\" class=\"pid\">10</a></li>\n</ul>\n<p>\n  <i id=\"p_11\" class=\"pid\"></i>Reach makes no attempt to ensure that when the application ends, all boxes are freed.\n  You should read about the <span class=\"snip\"><a href=\"/rsh/appinit/#rsh_ALGOExitMode\" title=\"rsh: ALGOExitMode\"><span style=\"color: var(--shiki-color-text)\">ALGOExitMode</span></a></span> option for <span class=\"snip\"><a href=\"/rsh/appinit/#rsh_setOptions\" title=\"rsh: setOptions\"><span style=\"color: var(--shiki-color-text)\">setOptions</span></a></span> for more information about the issues this poses.<a href=\"#p_11\" class=\"pid\">11</a>\n</p>\n<p>\n  <i id=\"p_12\" class=\"pid\"></i>In Algorand, network time corresponds to round numbers.\n  Each round is assigned a Unix timestamp, but when you look at the timestamp in code executing in round N+1, you read the timestamp assigned to round N.\n  Ensure that you read <a href=\"/rsh/errors/#RW0006\">RW0006</a> if you use network seconds.<a href=\"#p_12\" class=\"pid\">12</a>\n</p>\n<p><i id=\"p_13\" class=\"pid\"></i>The connector provides a binding named <span class=\"snip\"><a href=\"/rsh/appinit/#rsh_ALGO\" title=\"rsh: ALGO\"><span style=\"color: var(--shiki-token-constant)\">ALGO</span></a></span> to backends.<a href=\"#p_13\" class=\"pid\">13</a></p>\n<p><i id=\"p_14\" class=\"pid\"></i>Reach uses the following environment variables:<a href=\"#p_14\" class=\"pid\">14</a></p>\n<ul>\n  <li>\n    <i id=\"p_15\" class=\"pid\"></i><code>ALGO_TOKEN</code> is used as the API token for your <code>algod</code>.\n    When left unspecified, this defaults to the token of the Reach devnet.<a href=\"#p_15\" class=\"pid\">15</a>\n  </li>\n  <li>\n    <i id=\"p_16\" class=\"pid\"></i><code>ALGO_TOKEN_HEADER</code> is used as the HTTP header to share the <code>algod</code> API token.\n    When left unspecified, this defaults to <code>X-Algo-API-Token</code>.<a href=\"#p_16\" class=\"pid\">16</a>\n  </li>\n  <li>\n    <i id=\"p_17\" class=\"pid\"></i><code>ALGO_SERVER</code> is used as the address of your <code>algod</code>.\n    When left unspecified, this defaults to <code>http://localhost</code>.<a href=\"#p_17\" class=\"pid\">17</a>\n  </li>\n  <li>\n    <i id=\"p_18\" class=\"pid\"></i><code>ALGO_PORT</code> is used as the port of your <code>algod</code>.\n    When left unspecified, this defaults to <code>4180</code>.<a href=\"#p_18\" class=\"pid\">18</a>\n  </li>\n  <li>\n    <i id=\"p_19\" class=\"pid\"></i><code>ALGO_INDEXER_TOKEN</code> is used as the API token for your <code>indexer</code>.\n    When left unspecified, this defaults to the token of the Reach devnet.<a href=\"#p_19\" class=\"pid\">19</a>\n  </li>\n  <li>\n    <i id=\"p_20\" class=\"pid\"></i><code>ALGO_INDEXER_TOKEN_HEADER</code> is used as the HTTP header to share the <code>indexer</code> API token.\n    When left unspecified, this defaults to <code>X-Indexer-API-Token</code>.<a href=\"#p_20\" class=\"pid\">20</a>\n  </li>\n  <li>\n    <i id=\"p_21\" class=\"pid\"></i><code>ALGO_INDEXER_SERVER</code> is used as the address of your <code>indexer</code>.\n    When left unspecified, this defaults to <code>http://localhost</code>.<a href=\"#p_21\" class=\"pid\">21</a>\n  </li>\n  <li>\n    <i id=\"p_22\" class=\"pid\"></i><code>ALGO_INDEXER_PORT</code> is used as the port of your <code>indexer</code>.\n    When left unspecified, this defaults to <code>8980</code>.<a href=\"#p_22\" class=\"pid\">22</a>\n  </li>\n  <li>\n    <i id=\"p_23\" class=\"pid\"></i><code>ALGO_NODE_WRITE_ONLY</code> specifies if your <code>algod</code> will handle read requests,\n    or just writes.\n    When left unspecified, this defaults to <code>no</code>.<a href=\"#p_23\" class=\"pid\">23</a>\n  </li>\n  <li>\n    <i id=\"p_24\" class=\"pid\"></i><code>ALGO_FAUCET_PASSPHRASE</code> is used as the mnemonic for the faucet of your network.\n    When left unspecified, this defaults to the passphrase of the Reach devnet's faucet.\n    This is useful if you are running your own testing network.<a href=\"#p_24\" class=\"pid\">24</a>\n  </li>\n  <li>\n    <i id=\"p_25\" class=\"pid\"></i><code>ALGO_GENESIS_ID</code> or <code>ALGO_GENESIS_HASH</code> (or both),\n    which allows you to request that the user's <a href=\"https://github.com/algorandfoundation/ARCs/blob/main/ARCs/arc-0006.md\" target=\"_blank\">ARC-6 compliant wallet</a> connect to a specific network.\n    When left unspecified, it allows the user to select one of their wallet's supported networks.<a href=\"#p_25\" class=\"pid\">25</a>\n  </li>\n  <li>\n    <i id=\"p_26\" class=\"pid\"></i><code>ALGO_ACCOUNT</code>,\n    which allows you to request the use of a specific account from the user's ARC-6 compliant wallet by address.\n    This should usually be left unspecified, which allows the user to instead select their preferred account.<a href=\"#p_26\" class=\"pid\">26</a>\n  </li>\n</ul>\n<h2 id=\"ref-network-eth\" class=\"refHeader\">Ethereum<a aria-hidden=\"true\" tabindex=\"-1\" href=\"#ref-network-eth\"><span class=\"icon icon-link\"></span></a></h2>\n<p>\n  <i id=\"p_27\" class=\"pid\"></i>The <a href=\"https://ethereum.org/\" target=\"_blank\">Ethereum</a> Reach connector generates a contract that\n  manages one instance of the DApp's execution.\n  It is guaranteed to\n  use exactly one word of on-chain state, while each piece of consensus state appears as a transaction argument.<a href=\"#p_27\" class=\"pid\">27</a>\n</p>\n<p>\n  <i id=\"p_28\" class=\"pid\"></i>Ethereum uses the Keccak256 algorithm to perform digests.\n  Its bit width is 256-bits.<a href=\"#p_28\" class=\"pid\">28</a>\n</p>\n<p>\n  <i id=\"p_29\" class=\"pid\"></i>Non-network tokens are compiled to <a href=\"https://ethereum.org/en/developers/docs/standards/tokens/erc-20/\" target=\"_blank\">ERC-20</a> fungible tokens.\n  Specifically, the <span class=\"snip\"><a href=\"/rsh/compute/#rsh_Token\" title=\"rsh: Token\"><span style=\"color: var(--shiki-color-text)\">Token</span></a></span> type refers to the address of the ERC-20 contract.\n  Token minting launches a fresh ERC-20 contract based on the OpenZeppelin ERC-20 implementation, which stores additional metadata and allows the creator to burn tokens and destroy the token if there is no supply (i.e. it has all been burned).<a href=\"#p_29\" class=\"pid\">29</a>\n</p>\n<p>\n  <i id=\"p_30\" class=\"pid\"></i>Views are compiled to <code>view</code> functions.\n  A view named <code>X.Y</code> will be named <code>X_Y</code>.\n  A view named <code>X</code> will be named <code>X</code>.\n  Views expand the on-chain state to include the free variables of all values bound to a view.<a href=\"#p_30\" class=\"pid\">30</a>\n</p>\n<p>\n  <i id=\"p_31\" class=\"pid\"></i>In Ethereum, network time corresponds to block numbers and network seconds correspond to the Unix timestamp of the block.\n  Ensure that you read <a href=\"/rsh/errors/#RW0006\">RW0006</a> if you use network seconds.<a href=\"#p_31\" class=\"pid\">31</a>\n</p>\n<p>\n  <i id=\"p_32\" class=\"pid\"></i>The connector provides a binding named <span class=\"snip\"><a href=\"/rsh/appinit/#rsh_ETH\" title=\"rsh: ETH\"><span style=\"color: var(--shiki-token-constant)\">ETH</span></a></span> to\n  backends.<a href=\"#p_32\" class=\"pid\">32</a>\n</p>\n<p>\n  <i id=\"p_33\" class=\"pid\"></i>During compilation, the connector produces one intermediate output: <code>input.export.sol</code>, containing\n  the Solidity code implementing the contract.<a href=\"#p_33\" class=\"pid\">33</a>\n</p>\n<p>\n  <i id=\"p_34\" class=\"pid\"></i>A few details of Ethereum leak through to Reach.\n  The node that a given participant is connected to does not instantly know that its blocks are correct and may revert past transactions after it reaches consensus with the rest of the network.\n  This means that Reach applications must not make externally observable effects until after such consensus is reached.<a href=\"#p_34\" class=\"pid\">34</a>\n</p>\n<p><i id=\"p_35\" class=\"pid\"></i>Backends must respect the following environment variables:<a href=\"#p_35\" class=\"pid\">35</a></p>\n<ul>\n  <li>\n    <i id=\"p_36\" class=\"pid\"></i><code>ETH_NODE_URI</code> is used to contact the Ethereum node.\n    It defaults to <code>http://localhost:8545</code>.<a href=\"#p_36\" class=\"pid\">36</a>\n  </li>\n  <li>\n    <i id=\"p_37\" class=\"pid\"></i><code>ETH_NODE_NETWORK</code> is used to name the Ethereum network.\n    It defaults to <code>unspecified</code>.<a href=\"#p_37\" class=\"pid\">37</a>\n  </li>\n</ul>","<ul><li class=\"dynamic\">\n    <a href=\"#ref-networks\">Networks</a>\n    <ul class=\"dynamic\">\n
